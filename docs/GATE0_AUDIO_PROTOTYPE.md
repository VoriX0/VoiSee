# Gate 0 Audio Prototype

## Цель

Проверить, что VoiSe может в реальном времени захватывать физический микрофон, обрабатывать сигнал, выводить его в VB-CABLE и мониторить итоговый звук в наушниках.

## Проверяемый тракт

```text
Physical microphone
-> WasapiCapture
-> float32 processing
-> gate / compressor / gain / limiter
-> BufferedWaveProvider
-> WasapiOut to VB-CABLE Input
-> VB-CABLE Output as microphone in Discord/Telegram
```

## Критерии успешности

1. Прототип видит физический микрофон.
2. Прототип видит `CABLE Input` как устройство вывода.
3. Голос попадает в `CABLE Output`.
4. Discord/Telegram видят `CABLE Output` как микрофон.
5. Мониторинг работает в выбранных наушниках.
6. Нет постоянного треска/обрывов при обычной речи.
7. Stop all / Ctrl+C корректно останавливает аудиотракт.
8. При отключении микрофона приложение не падает без сообщения об ошибке.

## Что не входит в Gate 0

- WinUI 3 интерфейс.
- Сцены.
- SoundBoard UI.
- OGG-декодер.
- полноценный pitch/formant engine.
- шумоподавление.

## Ручной замер задержки

1. Запустить прототип с выводом в VB-CABLE.
2. Включить запись в Audacity/OBS/любом рекордере, который пишет `CABLE Output`.
3. Подать короткий щелчок рядом с микрофоном.
4. Зафиксировать задержку между исходным импульсом и сигналом после VB-CABLE.
5. Значение до 80–120 мс считается допустимым для MVP.
