# VoiSee 11 — Media Bridge

## 1. Назначение

VoiSee 11 добавляет универсальный **Media Bridge**: захват звука выбранного окна или приложения и передачу этого звука в виртуальный микрофон. Яндекс Музыка является первым целевым сценарием, но аудиомаршрут не зависит от API конкретного сервиса.

Media Bridge не заменяет микрофон, SoundBoard, Voice Changer или звуки сцен и работает как отдельный источник общего микшера.

## 2. Отдельная вкладка

Основная область вкладки делится на две панели:

- слева — крупный preview выбранного окна;
- справа — управление, сведения об источнике и настройки трансляции.

Preview всегда вписывает изображение окна целиком без обрезания. Нажатие на сам preview открывает выбор окна. Пока источник не выбран, по центру показывается крупный `+`. После выбора остаётся компактный центральный `+`, показывающий, что источник можно заменить нажатием на preview.

Правая панель содержит:

- кнопку `Play / Pause`;
- кнопку `Stop`, которая отсутствует до выбора окна;
- название окна и процесса;
- статус;
- длительность текущей трансляции;
- вертикальный индикатор уровня исходного сигнала;
- Volume in Microphone;
- будущие Normalization и Ducking.

## 3. Поведение управления

### Play / Pause

Кнопка имеет три состояния:

- `Play` при выбранном, но ещё не запущенном источнике — начинает захват;
- `Pause` во время трансляции — временно заглушает только Media Bridge;
- `Play` в состоянии Paused — возвращает поток в virtual mic.

Исходное приложение продолжает воспроизводить музыку во время Pause.

### Stop

Stop полностью завершает захват, очищает runtime-источник и preview. Сохранённое описательное имя профиля остаётся, но для нового запуска окно нужно выбрать повторно.

В будущей интеграции со сценами кнопка `Stop Broadcast` вызывает ровно ту же операцию Stop.

## 4. Аудиомаршрут

```text
Microphone ────────┐
SoundBoard ────────┼──► final limiter ─► Virtual Microphone
Scene sounds ──────┤
Media Bridge ──────┘
```

Требования:

- Media Bridge не направляется в Headphones;
- запуск и остановка Media Bridge не останавливают SoundBoard или сцену;
- сцена и SoundBoard не сбрасывают Media Bridge;
- общий limiter защищает итоговую сумму от клиппинга.

## 5. Настройки

### VoiSee 11.0

- Volume in Microphone;
- Play / Pause;
- Stop;
- вертикальный level indicator.

Индикатор показывает уровень звука, получаемого из выбранного окна до применения пользовательской громкости Media Bridge.

### VoiSee 11.1

- Normalization;
- один слайдер `Ducking`.

Слайдер Ducking одновременно включает эффект и задаёт силу приглушения:

- 0% — Ducking выключен;
- увеличение значения сильнее уменьшает громкость Media Bridge, когда пользователь говорит.

Отдельные настройки Ducking Strength, Attack и Release пользователю не показываются. Fade In и Fade Out в Media Bridge не добавляются.

## 6. Горячие клавиши

Добавляется только глобальное действие:

- Pause / Resume Media Bridge.

Глобальная горячая клавиша Stop не добавляется.

## 7. Поведение после перезапуска

VoiSee сохраняет:

- последнее описательное имя процесса и окна;
- громкость;
- будущие настройки normalization и ducking;
- Pause / Resume hotkey.

VoiSee не сохраняет PID или HWND, не ищет окно автоматически и не возобновляет трансляцию после перезапуска. Пользователь выбирает окно заново.

## 8. Интеграция со сценами — VoiSee 11.2

Фон сцены получает два взаимоисключающих режима:

1. Looped Sound;
2. Media Source.

При выборе Media Source:

- на панели указывается имя профиля приложения;
- HP slider скрывается;
- Mic slider остаётся;
- четыре кнопки looped sound заменяются на `Launch Source` и `Stop Broadcast`;
- чекбокс меняется на `Capture source when scene starts`.

`Launch Source` открывает установленное приложение, а при его отсутствии — browser fallback. Само нажатие не начинает захват. Захват начинается при применении сцены, если включён соответствующий чекбокс.

Если сцена сама начала трансляцию, она может остановить её при отключении. Ручной Media Bridge, работавший до применения сцены, сцена не останавливает.

## 9. Этапы

### 11.0 — Media Bridge Core

- отдельная вкладка;
- выбор окна нажатием на preview;
- preview без обрезания;
- один process-loopback источник;
- независимый media bus;
- Play, Pause, Resume, Stop;
- громкость и вертикальный level meter;
- сохранённый описательный профиль;
- Pause / Resume hotkey.

### 11.1 — Audio Processing

- normalization;
- один Ducking slider;
- обработка потери источника.

### 11.2 — Scene Media Background

- Loop / Media background;
- Launch Source и browser fallback;
- Capture source when scene starts;
- общая операция Stop;
- корректное владение трансляцией.

### 11.3 — Release Hardening

- Яндекс Музыка;
- браузеры;
- Spotify/VLC;
- миграция сцен;
- установщик и документация.
